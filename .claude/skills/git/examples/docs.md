# docs — examples

Use when only documentation or markdown files change — no logic, no data.

## Good

```
docs: update CONSTITUTION era progression rules
docs: add how-to-play section to README
docs: clarify voting period in CONSTITUTION Article V
docs: add GitHub Pages link to landing page footer
```

## Bad

```
docs: update docs              ← too vague
docs: README                   ← not imperative
Docs update                    ← missing type prefix
```
